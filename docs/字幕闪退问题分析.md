# 字幕插入导致达芬奇闪退问题分析

## 🔴 问题现象
- 音频正确插入 ✅
- 字幕未插入 ❌
- 达芬奇闪退 💥

## 🔍 根本原因

### 对比音频和字幕的 clipInfo 实现

#### 音频插入（正常工作）
```python
clip_info = {
    "mediaPoolItem": selected_clip,
    "startFrame": 0,
    "endFrame": clip_duration_frames - 1,      # ← 注意：减 1
    "trackIndex": track_index,                 # ← 有轨道索引
    "recordFrame": start_frame,  
    "stereoEye": "both"                        # ← 有立体声设置
}
```

#### 字幕插入（导致闪退）
```python
clip_info = {
    "mediaPoolItem": subtitle_clip,
    "startFrame": 0,
    "endFrame": end_frame - start_frame,       # ❌ 没有减 1
    "recordFrame": start_frame,                # ❌ 缺少 trackIndex
}                                              # ❌ 缺少其他参数
```

## 🐛 具体问题

### 1. **缺少 `trackIndex` 参数**
- **音频**：明确指定了轨道索引（通过 `get_first_empty_track` 查找）
- **字幕**：没有指定轨道，API 不知道插入到哪个字幕轨道
- **后果**：可能导致 API 调用失败或访问无效内存 → 闪退

### 2. **`endFrame` 计算错误**
- **音频**：`clip_duration_frames - 1`（正确的帧范围）
- **字幕**：`end_frame - start_frame`（可能超出范围）
- **后果**：帧范围不正确，可能导致越界访问

### 3. **缺少字幕轨道查找逻辑**
音频有专门的轨道查找函数：
```python
track_index = get_first_empty_track(current_timeline, start_frame, end_frame, "audio")
```

字幕没有对应的轨道查找，直接插入可能冲突。

## 💡 解决方案

### 方案 1：添加字幕轨道索引（推荐）
```python
# 1. 查找或创建字幕轨道
track_index = get_first_empty_track(current_timeline, start_frame, end_frame, "subtitle")

# 2. 修正 clipInfo
clip_info = {
    "mediaPoolItem": subtitle_clip,
    "startFrame": 0,
    "endFrame": end_frame - start_frame - 1,   # ← 修正：减 1
    "trackIndex": track_index,                 # ← 添加：轨道索引
    "recordFrame": start_frame,
}
```

### 方案 2：使用字幕专用 API（如果存在）
检查 DaVinci Resolve API 是否有专门的字幕插入方法，而不是使用通用的 `AppendToTimeline`。

### 方案 3：简化为直接插入（回退方案）
```python
# 不使用 clipInfo，直接插入
success = media_pool.AppendToTimeline([subtitle_clip])
```
但这样会失去位置控制。

## 🎯 推荐修复步骤

1. **检查 `get_first_empty_track` 函数是否支持字幕轨道**
2. **修正 `endFrame` 计算（减 1）**
3. **添加 `trackIndex` 参数**
4. **测试是否还需要其他参数（如 `stereoEye`）**
