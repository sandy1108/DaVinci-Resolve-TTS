# MiniMax API 字幕断句问题分析

## 问题描述

用户反馈：当使用 MiniMax API 生成字幕时，即使输入了很长的句子，返回的字幕也没有自动断句，所有文本都显示在同一个时间点上，导致字幕同时显示所有内容，不符合实际使用需求。

## 官方文档说明

根据 MiniMax 官方 API 文档（`/v1/t2a_v2` 接口）：

### 字幕功能参数

**请求参数：**
- **subtitle_enable** (boolean, 默认 false)
  - 控制是否开启字幕服务
  - 仅在**非流式输出**场景下有效
  - 支持模型：`speech-2.6-hd`, `speech-2.6-turbo`, `speech-02-hd`, `speech-02-turbo`, `speech-01-hd`, `speech-01-turbo`

**响应参数：**
- **data.subtitle_file** (string)
  - 合成的字幕下载链接
  - **格式说明：** 音频文件对应的字幕，**精确到句（不超过 50 字）**，单位为毫秒，格式为 JSON

## 关键发现

### ✅ API 支持自动断句

根据官方文档明确说明：

> **"精确到句（不超过 50 字）"**

这表明 MiniMax API **确实支持自动断句功能**，并且有以下特点：

1. **自动分句：** API 会自动将长文本分成多个句子
2. **长度限制：** 每个字幕片段不超过 50 字
3. **时间精确：** 每个句子都有对应的时间戳（毫秒级）
4. **JSON 格式：** 字幕以 JSON 格式返回

### 字幕文件格式（推测）

虽然官方文档没有给出具体的 JSON 格式示例，但根据描述，字幕文件应该类似以下结构：

```json
[
  {
    "text": "这是第一句话，不超过五十个字。",
    "start_time": 0,
    "end_time": 2500
  },
  {
    "text": "这是第二句话，也会自动断句。",
    "start_time": 2500,
    "end_time": 5000
  },
  {
    "text": "即使输入很长的文本，API也会自动分割。",
    "start_time": 5000,
    "end_time": 7800
  }
]
```

或者可能是 SRT/WebVTT 类似的格式。

## 可能的问题原因

### 1. 字幕文件未正确下载和解析

**问题：** 项目代码可能只获取了 `subtitle_file` URL，但没有：
- 下载字幕文件内容
- 解析 JSON 格式的字幕数据
- 将字幕按时间轴正确导入到 DaVinci Resolve

**代码检查点：**
```python
# 在 synthesize() 方法中
resp_data = response.json()
data = resp_data.get("data", {})

# 返回了 subtitle_url
return {
    "audio_content": bytes.fromhex(audio_hex), 
    "subtitle_url": data.get("subtitle_file"),  # ← 只返回了URL
    ...
}
```

**可能的问题：**
- 代码只返回了 URL，但没有下载字幕文件
- 没有解析字幕 JSON 数据
- 没有将字幕数据转换为 DaVinci Resolve 可用的格式

### 2. 字幕导入逻辑缺失

即使获取了字幕 URL，也需要：
1. 下载字幕文件内容
2. 解析 JSON 格式
3. 将每个字幕片段按时间轴添加到 DaVinci Resolve 时间线

### 3. 文本输入方式问题

**可能影响因素：**
- 如果输入文本没有使用标点符号，API 可能难以正确断句
- 建议在输入文本时使用适当的标点符号（句号、问号、感叹号等）

## 解决方案建议

### 方案 1：完善字幕下载和解析（推荐）

1. **下载字幕文件：**
```python
def download_subtitle(self, subtitle_url: str) -> List[Dict]:
    """下载并解析字幕文件"""
    if not subtitle_url:
        return []
    
    try:
        response = requests.get(subtitle_url, timeout=60)
        response.raise_for_status()
        subtitle_data = response.json()
        return subtitle_data
    except Exception as e:
        print(f"Failed to download subtitle: {e}")
        return []
```

2. **解析字幕数据：**
   - 获取每个字幕片段的文本、开始时间、结束时间
   - 转换时间格式（毫秒 → 帧数）

3. **导入到 DaVinci Resolve：**
   - 在时间线上创建字幕轨道
   - 按时间轴添加每个字幕片段

### 方案 2：使用文本断句标记

在输入文本时，使用 MiniMax 支持的停顿控制标记：

```python
# 使用 <#x#> 标记手动控制断句和停顿
text = "这是第一句话。<#0.5#>这是第二句话。<#0.5#>这是第三句话。"
```

**说明：**
- `<#x#>` 中的 x 为停顿时长（秒）
- 范围：[0.01, 99.99]
- 这样可以手动控制断句点

### 方案 3：预处理文本

在发送到 API 之前，先对文本进行预处理：

```python
def split_text_by_sentences(text: str, max_length: int = 50) -> List[str]:
    """将长文本按句子分割，每句不超过 max_length 字符"""
    import re
    
    # 按标点符号分句
    sentences = re.split(r'([。！？.!?])', text)
    
    result = []
    current = ""
    
    for i in range(0, len(sentences), 2):
        sentence = sentences[i]
        punctuation = sentences[i+1] if i+1 < len(sentences) else ""
        
        if len(current + sentence + punctuation) <= max_length:
            current += sentence + punctuation
        else:
            if current:
                result.append(current)
            current = sentence + punctuation
    
    if current:
        result.append(current)
    
    return result
```

然后分别调用 API 生成每句的音频和字幕。
