### 🔥 完整版独有功能

根据代码分析，完整版（3.3.py）包含以下独有功能：

#### 1. MiniMax 音色快速复刻（Voice Cloning）

**核心类：**
```python
class MiniMaxProvider:
    def upload_file_for_clone(self, file_path: str)
    def submit_clone_job(self, file_id, voice_id, need_nr, need_vn, text)
    def download_media(self, url: str)
```

**功能特点：**
- 🎤 **从 DaVinci Resolve 获取音频** - 使用 Marker 标记音频范围，直接从时间线渲染音频
- 📤 **上传音频到 MiniMax** - 自动上传音频文件进行音色复刻
- 🎨 **创建自定义音色** - 生成专属的 voice_id
- 🔊 **试听复刻效果** - 支持试听复刻后的音色
- 💾 **音色管理** - 保存和管理复刻的音色 ID
- 🎯 **降噪和音量归一化** - 优化音频质量

**使用流程：**
1. 在时间线上用 Marker 标记要复刻的音频片段（10秒-5分钟）
2. 点击"上传音频"按钮
3. DaVinci Resolve 自动渲染该片段
4. 上传到 MiniMax API
5. 输入自定义 voice_id
6. 可选：上传示例音频增强效果
7. 可选：输入试听文本
8. 提交复刻任务
9. 获取复刻的 voice_id，可在语音合成中使用

**相关 UI 组件：**
- 音色复刻配置窗口
- 上传音频按钮
- 降噪选项
- 音量归一化选项
- 试听文本输入框
- voice_id 输入框

#### 2. 更完善的 MiniMax API 封装

完整版包含完整的 `MiniMaxProvider` 类，提供：
- ✅ 完整的错误处理机制

## 技术细节

### MiniMax 音色复刻 API 调用流程

完整版实现的音色复刻流程：

```python
# 1. 渲染音频
audio_file = render_audio_by_marker(output_dir)

# 2. 上传音频文件
provider = MiniMaxProvider(api_key, group_id)
result = provider.upload_file_for_clone(audio_file)
file_id = result["file_id"]

# 3. 提交复刻任务
clone_result = provider.submit_clone_job(
    file_id=file_id,
    voice_id="custom_voice_001",
    need_nr=True,          # 降噪
    need_vn=True,          # 音量归一化
    text="试听文本",        # 可选
    model="speech-02-hd"   # 可选
)

# 4. 获取试听音频
demo_url = clone_result["demo_audio"]
audio_content = provider.download_media(demo_url)

# 5. 保存 voice_id 供后续使用
save_voice_id("custom_voice_001")
```

### 音频要求

**待复刻音频：**
- 格式：mp3, m4a, wav
- 时长：10秒 - 5分钟
- 大小：≤ 20MB

**示例音频（可选）：**
- 格式：mp3, m4a, wav
- 时长：< 8秒
- 大小：≤ 20MB

## 总结

**核心区别：**

1. **免费版** = 基础 TTS 功能 + 系统音色
2. **完整版** = 免费版 + 音色快速复刻 + 高级功能

**最大的差异就是音色复刻功能**，这是一个专业级的功能，允许用户：
- 从自己的音频创建独特的 AI 音色
- 复刻特定角色或人物的声音
- 用于商业项目的 IP 音色制作

如果你只是需要基础的文字转语音功能，免费版完全够用。如果需要创建自定义音色，就需要使用完整版。
