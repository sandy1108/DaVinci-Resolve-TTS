# 字幕闪退问题修复说明

## 🔴 问题回顾
- **现象**：音频正确插入，但字幕未插入且导致达芬奇闪退
- **原因**：字幕插入的 `clipInfo` 缺少关键参数，与音频插入逻辑不一致

## 🔧 修复内容

### 修改位置
**文件**：`DaVinci Resolve TTS/DaVinci TTS 3.3.py`  
**行数**：1056-1077 行

### 核心修复

#### 修复前（导致闪退）❌
```python
subtitle_clip = imported[0]

clip_info = {
    "mediaPoolItem": subtitle_clip,
    "startFrame": 0,
    "endFrame": end_frame - start_frame,     # ❌ 没有减 1
    "recordFrame": start_frame,              # ❌ 缺少 trackIndex
}

success = media_pool.AppendToTimeline([clip_info])
```

#### 修复后（完整实现）✅
```python
subtitle_clip = imported[0]

# 1. 查找第一个空的字幕轨道
track_index = get_first_empty_track(timeline, start_frame, end_frame, "subtitle")

# 2. 创建完整的 clipInfo
subtitle_duration_frames = end_frame - start_frame
clip_info = {
    "mediaPoolItem": subtitle_clip,
    "startFrame": 0,
    "endFrame": subtitle_duration_frames - 1,  # ✅ 修正：减 1
    "trackIndex": track_index,                 # ✅ 添加：字幕轨道索引
    "recordFrame": start_frame,
}

success = media_pool.AppendToTimeline([clip_info])
```

## 🎯 修复要点

### 1. 添加 `trackIndex` 参数
- **作用**：指定字幕插入到哪个轨道
- **实现**：使用 `get_first_empty_track(timeline, start_frame, end_frame, "subtitle")` 查找空轨道
- **重要性**：⭐⭐⭐⭐⭐ 缺少此参数会导致 API 不知道插入位置，可能访问无效内存导致闪退

### 2. 修正 `endFrame` 计算
- **修改**：`end_frame - start_frame` → `subtitle_duration_frames - 1`
- **原因**：帧范围是从 0 开始的，需要减 1（例如：5 秒 = 150 帧，范围是 0-149）
- **参考**：音频插入使用 `clip_duration_frames - 1`

### 3. 与音频插入逻辑保持一致
现在字幕和音频的插入逻辑完全对齐：

| 参数 | 音频 | 字幕（修复后） |
|------|------|----------------|
| `mediaPoolItem` | ✅ | ✅ |
| `startFrame` | ✅ 0 | ✅ 0 |
| `endFrame` | ✅ duration - 1 | ✅ duration - 1 |
| `trackIndex` | ✅ 查找空轨道 | ✅ 查找空轨道 |
| `recordFrame` | ✅ 插入位置 | ✅ 插入位置 |

## 📝 技术细节

### `get_first_empty_track` 函数
```python
def get_first_empty_track(timeline, start_frame, end_frame, media_type):
    """获取当前播放头位置的第一个空轨道索引"""
    track_index = 1
    while True:
        items = timeline.GetItemListInTrack(media_type, track_index)
        if not items:
            return track_index
        
        # 检查轨道上是否有片段与给定的start_frame和end_frame重叠
        is_empty = True
        for item in items:
            if item.GetStart() <= end_frame and start_frame <= item.GetEnd():
                is_empty = False
                break
        
        if is_empty:
            return track_index
        
        track_index += 1
```

该函数支持的 `media_type`：
- `"audio"` - 音频轨道
- `"video"` - 视频轨道
- `"subtitle"` - 字幕轨道
